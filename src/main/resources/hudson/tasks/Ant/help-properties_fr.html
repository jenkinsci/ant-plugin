Vous pouvez indiquer ici les propriétés nécessaires pour votre build Ant (avec le format standard des fichiers de propriétés) : 
# commentaire
nom1=valeur1
nom2=$VAR2
Ces propriétés sont passées à Ant ainsi : "-Dnom1=valeur1 -Dnom2=valeur2". Utilisez toujours la forme $VAR (même sur Windows) pour les références aux variable d'environnement définies dans Jenkins. Sur Windows, les références de la forme %VAR% peuvent être utilisées pour les variables d'environnement qui existent en dehors de Jenkins. Les anti-slashs sont utilisés pour l'échappement, ainsi utiliser \\ pour un seul anti-slash. Les double quotes (") doivent être évités, car ant sur *nix entoure les paramètres de quotes et les passe dans eval, et Windows a ses propres problèmes d'échappement. Dans chaque cas, l'usage des quotes peut entraîner des échecs de build. Pour définir une propriété vide, écrire simplement varname=
